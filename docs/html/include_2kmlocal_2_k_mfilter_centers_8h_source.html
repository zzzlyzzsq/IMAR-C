00026 //----------------------------------------------------------------------
00027 //  KMfilterCenters
00028 //      This extends the KMcenters class, by providing a more efficient
00029 //      algorithm for computing distortions, by the filtering algorithm.
00030 //      This algorithm makes use of a data structure called a kc-tree,
00031 //      which is given in the file KCtree.h.  In addition to the
00032 //      KMcenter functions we provide the following additional
00033 //      functions:
00034 //
00035 //      getDist()
00036 //      getDists()
00037 //              Computes the total and individual distortions,
00038 //              respectively, for the centers points (see definitions
00039 //              below).
00040 //      moveToCentroid()
00041 //              Moves the center points to the centroids of their
00042 //              associated neighborhoods.
00043 //      getAssignments()
00044 //              Computes the assignment of points to the closest center.
00045 //
00046 //      These functions are not computed independently.  In particular,
00047 //      for a given set of centers, they can each be computed very
00048 //      efficiently (in O(k*d) time) provided that some intermediate
00049 //      values has already been computed.  We maintain a status variable
00050 //      "valid," which indicates whether these intermediate values have
00051 //      been computed and are current.  The intermediate values are
00052 //      computed by computeDistortion().
00053 //
00054 //      >> If you modify this class note that any function that <<
00055 //      >> modifies the center or data points must set run      <<
00056 //      >> invalidate() or equivalently, set valid=false.       <<
00057 //
00058 //      Immediate access:
00059 //      -----------------
00060 //      To disable the automatic recomputation of distortions on
00061 //      getDist() and getDists(), call them with a "false" argument.
00062 //
00063 //      Distortion Overview:
00064 //      --------------------
00065 //      Let C[j] denote the j-th center.  For each j in [0..k-1], define
00066 //      the j-th neighborhood V(j), to be the set of data points that
00067 //      are closer to j than to any other center.  The "j-th distortion"
00068 //      is defined to be the sum of squared distances of every point in
00069 //      V(j) to the j-th center.  The "total distortion" is the sum of
00070 //      the distortions over all the centers.
00071 //
00072 //      Intermediate Values:
00073 //      --------------------
00074 //      Instead of computing distortions from scratch by brute force
00075 //      (which would take O(n*k*d) time), we use an algorithm called the
00076 //      filtering algorithm.  This algorithm does not compute the
00077 //      distortion directly, but instead computes the following
00078 //      intermediate values, from which the distortion can be computed
00079 //      efficiently.  Let j be an index in [0..k-1].  The notation (u.v)
00080 //      denotes the dot product of vectors u and v.
00081 //
00082 //      KMpoint sums[j]         Vector sum of points in V(j)
00083 //      double sumsSqs[j]       Sum of (u.u) for all u in V(j)
00084 //      double weights[j]       Number of data points such that
00085 //                                this C[j] is closest
00086 //
00087 //      See the function computeDistortion() and moveToCentroid() for
00088 //      explanations of how these quantities are combined to compute
00089 //      the total distortion and move centers to their centroids.
00090 //
00091 //      Final Values:
00092 //      -------------
00093 //      Given the above intermediate values, we then compute the
00094 //      following final distortion values.
00095 //
00096 //      double dists[j]         Total distortion for points of V(j)
00097 //      double currDist         Current total distortion
00098 //
00099 //      Although they are not used by this program, the center
00100 //      distortions are useful, because they may be used in a more
00101 //      general clustering algorithm to determine whether clusters
00102 //      should be split or merged.
00103 //----------------------------------------------------------------------
