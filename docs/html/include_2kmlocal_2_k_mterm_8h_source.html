00026 //------------------------------------------------------------------------
00027 //  KMterm - termination condition
00028 //      This structure is used for storing information controlling the
00029 //      termination and phase changes of the various algorithms.
00030 //
00031 //      Maximum Total Stages:
00032 //      ---------------------
00033 //      The algorithm is terminated after some maximum number of stages
00034 //      have been performed.  Rather than computing this as a fixed
00035 //      constant, it is given as a function of n (number of data points)
00036 //      and k (number of centers).  We use the following formula, where
00037 //      the coefficients a, ..., d are supplied by the user (see
00038 //      kmltest.cpp).
00039 //
00040 //              MAX_STAGE = a + (b*k + c*n)^d
00041 //
00042 //      Parameters Determining Phase/Run Transitions:
00043 //      ---------------------------------------------
00044 //      The local improvement algorithms consist of a series of stages,
00045 //      which are grouped into "runs" and series of runs are grouped in
00046 //      "phases".  The meaning of these groupings depends on the
00047 //      particular algorithm. (See KMlocal.h for more information.)  The
00048 //      transition between runs and phases are either based on the
00049 //      number of stages performed or on the change in distortion over
00050 //      the course of the run.
00051 //
00052 //      maxRunStage
00053 //              This is used to limit the maximum number of stages in
00054 //              any run.
00055 //
00056 //      Some transitions are defined in terms of a quantity called the
00057 //      "relative distortion loss" (RDL), which is defined to be the
00058 //      relative decrease in the distortion.  (See KMlocal.h for
00059 //      definition.) The relative distortion loss between the current
00060 //      and previous stages is called the "consecutive RDL" and the
00061 //      relative distortion loss since the start of a run is called the
00062 //      "accumulated RDL".
00063 //
00064 //      minConsecRDL
00065 //              This is used in the hybrid's algorithm.  If the RDL of
00066 //              two consecutive runs is less than this value, Lloyd's
00067 //              algorithm is deemed to have converged.
00068 //      minAccumRDL
00069 //              This is used in run-based algorithms.  It is the RDL of
00070 //              the current distortion relative to the distortion at
00071 //              some prior time (e.g. the start of a run).
00072 //
00073 //      Parameters used in Simulated Annealing
00074 //      --------------------------------------
00075 //      initProbAccept
00076 //              Initial probability of accepting an solution that does
00077 //              not alter the distortion.
00078 //      tempRunLength
00079 //              The number of stages before chaning the temperature.
00080 //      tempReducFactor
00081 //              The factor by which temperature is reduced at the end of
00082 //              a temperature run.
00083 //------------------------------------------------------------------------
